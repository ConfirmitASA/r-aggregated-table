(static) getData(options) → {Array}

Source:
  • table-data.js, line 59

A universal data-extraction function. It strips data from a table's body. Data can be stripped by rows (horizontally) or by columns (vertically) which is controlled by direction. It accounts for a spanning block cell and may exclude it.

Parameters:
Name Type Description
options Object

options to configure the way data is stripped off the table

Properties
Name Type Attributes Default Description
source HTMLTableElement

source table that will be an input for data stripping
direction String <optional>
 'row'

direction in which data stripping will occur: row strips across rows and presents an array where each array item is an array of cell values. column strips values verticaly in a column, the resulting array will contain arrays (per column) with values resembling normalized data for cells in the column
excludeBlock Boolean <optional>
 true

if table contains block cells that rowspan across several rows, we might need to exclude those from actual data
excludeColumns Array | Number <optional>

if table contains columns that are not to be in data, then pass a single index or an array of cell indices (0-based). You need to count columns not by headers but by the cells in rows.
excludeRows Array | Number <optional>

if table contains rows that are not to be in data, then pass a single index or an array of row indices (0-based). You need to count only rows that contain data, not the table-header rows.
multidimensional Boolean <optional>
 false

whether the table has aggregating cells that aggregate rowheaders. Result of TableData#detectMultidimensional may be passed here to automatically calculate if it has aggregating cells.
Returns:

returns data array.
